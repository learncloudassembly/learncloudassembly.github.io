Deployments

You can access the current status of your provisioned resources.

You can access detailed information for the deployment, such as: who owns the deployment, requested it, which Cloud Template has been used, expiration, created on and last updated dates. Additionally, Day 2 operations that affects all the resources within the deployment:

  • Change Lease
  • Change Owner
  • Delete
  • Edit Tags
  • Power Off
  • Update
  • Change Project
  • Power On

The Deployment view allows you to see a visual representation of the deployment Topology, which includes also access to the resources assoicated properties. You can use the Topology tab to understand the deployment structure and resources.

History Tab includes all the provisioning events and any events related to actions that you run after requested item is deployed. If there are any problems with the provisioning process, the History tab events will help you with troubleshoot the failures.

Price Tab displays pricing information is based on vRealize Operations Manager or CloudHealth integrations. You can use the pricing card to understand how much your deployment is costing your organization:

Monitor Tab data provides information about the health of your deployment based on data from vRealize Operations Manager:

Optimize Tab provides utilization information about your deployment and offers suggestions for reclaiming or otherwise modifying the resources to optimize resource consumption. The optimization information is based on data from vRealize Operations Manager:

Deployments Search & Sorting Capabilities:

You can use search to locate deployments based on a broad range of values, not just the deployment name.

You can also locate deployments based on:

  • Name, description, and who created the deployment
  • Resource properties, such as resource name, IP address, image, region, tags, cloud account, or primary MAC address

When you use the search, observe the following advantages and constraints:

  • The search is not case sensitive
  • You can only search on initial strings, not embedded strings. For example, you can retrieve results for a network named MyResourceName using My or MyResource, but not ResourceName.
  • You can search for values that might exist in more than one value type. For example, you can search deployments that contain a specific resource name and a specific tag. The search format must include a space between the values. The space is an implied AND. For example, you can search for MyResourceName 192.0.2.0

If you are looking for projects or resource types, use the filter rather than the search. You can also use the filters with the search to locate deployments.

You also have the ability to Sort your deployment list by multiple criteria such as:

  • Name (ascending)
  • Name (descending)
  • Price (ascending)
  • Price (descending)
  • Created on (ascending)
  • Created on (descending)
  • Expires on (ascending)
  • Expires on (descending)
  • Last updated (ascending)
  • Last updated (descending)
